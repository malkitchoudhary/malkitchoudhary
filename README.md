## `01` &nbsp;// WHO I AM

<table width="100%">
<tr>
<td width="60%" valign="top">

I build machine learning systems the way an engineer builds infrastructure — not to make a notebook run once, but to make it survive real, messy data. That habit started with a query-priority classifier that kept leaking labels between train and test until I traced it three functions deep; it's the same instinct I now bring to every pipeline I touch.

I'm currently finishing my **BCA at Manipal University Jaipur** while training full-time in **Data Science & Machine Learning at Tips-G (RNS IT Solution)**. Most of my work lives in that overlap between NLP and recommendation systems — resume-to-job matching, paper recommendation via cosine similarity, movie recommenders served through FastAPI. I fuse text and numeric features with `scipy.sparse.hstack`, compare LightGBM against XGBoost instead of assuming one wins, and I'd rather explain *why* an LSTM gate does what it does than just import `Sequential` and hope.

Right now I'm pushing into **multimodal AI and Transformer internals** — the natural next step after months in RNN/LSTM territory. I compete on Kaggle not for leaderboard flexing but because real datasets break your assumptions in ways clean tutorials never do.
